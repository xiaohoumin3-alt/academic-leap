### 核心设计原则

1. **DFI (Data Flow Integrity) >= 0.99**
   - 所有事件通过唯一 eventId 追踪
   - 失败事件进入 Dead Letter Queue
   - 完整的事件链路记录

2. **学习有效性 (LE) 基础奖励**
   - 基于 LE > 15% 的成就奖励
   - 连续学习奖励叠加
   - 精准度额外奖励

3. **家长控制**
   - 每日练习时长限制
   - 每日题目数量限制
   - 排行榜显示开关
   - 学习时间限制

4. **不阻塞核心**
   - 游戏化作为观察者
   - 异步处理不影响主流程
   - 错误隔离不影响学习

## 系统集成 (System Integration)

### 学习流程

```
用户开始练习 → UOK推荐题目 → 用户作答 → RL诊断 → 
更新推荐 → 游戏化事件 → 更新用户属性 → 记录成就 → 
排行榜更新 → 家长报告生成
```

### 健康监控

```
RL健康检查 → 计算LE/CS/DFI → 判断系统状态 → 
健康/警告/危险/崩溃 → 相应降级行动
```

### 数据流

```
请求 → API Gateway → 处理逻辑 → 数据持久 → 
游戏化监听 → 事件处理 → 缓存更新 → 日志记录
```
