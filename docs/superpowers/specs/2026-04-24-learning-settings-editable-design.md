# 学习设置可编辑设计

**日期:** 2026-04-24
**状态:** 已批准

## 问题

用户选择教材后无法修改年级/科目/教材，导致用户被锁定在初始选择中。

## 目标

允许用户随时更改学习设置（年级/科目/教材），提供清晰的编辑入口。

## 设计方案

### UI 布局

```
┌─────────────────────────────────────────────┐
│ 🎓 学习设置              ✏️                 │  ← 新增编辑按钮
├─────────────────────────────────────────────┤
│ 8年级 · 数学          0/0 知识点            │
├─────────────────────────────────────────────┤
│ 学习进度: 0%  [━━━━━━━━━━━━]               │
├─────────────────────────────────────────────┤
│ [智能推荐]  [手动勾选]                       │
│                                             │
│ 内容区域...                                  │
└─────────────────────────────────────────────┘
```

### 交互流程

| 状态 | 行为 |
|------|------|
| **默认** | 显示当前设置，编辑按钮在标题右侧 |
| **点击编辑** | 展开教材选择器，标题变为"选择学习教材" |
| **选择完成** | 保存新设置，刷新知识树，返回默认状态 |
| **取消** | 关闭选择器，保持原设置 |

### 编辑按钮设计

- **图标**: `edit` 或 `pencil` (Material Icon)
- **位置**: 标题右侧，与标题同行
- **样式**: 小图标，点击有反馈

### 数据变更警告

当用户选择不同教材时，显示提示：

```
⚠️ 更换教材将清空当前的知识点选择，
需要重新勾选学习内容。

[取消]  [继续]
```

## 技术实现

### 状态管理

```typescript
const [isEditing, setIsEditing] = useState(false);
const [showWarning, setShowWarning] = useState(false);
const [pendingTextbookId, setPendingTextbookId] = useState<string | null>(null);
```

### 关键逻辑

1. **编辑模式切换**: `isEditing` 控制显示选择器还是设置面板
2. **教材变更检测**: 比较新旧 `selectedTextbookId`
3. **警告确认**: 用户确认后才执行更新

### API 调用

- 更新设置: `PUT /api/user/settings`
- 重新加载知识树: `GET /api/user/knowledge-tree`

## 边界情况

| 情况 | 处理 |
|------|------|
| 教材被删除 | 显示选择器，提示重新选择 |
| 加载失败 | 显示错误提示和重试按钮 |
| 用户取消编辑 | 恢复原状态，不保存更改 |

## 组件结构

```
LearningSettings
├── Header (标题 + 编辑按钮)
├── Summary (设置摘要)

    └── WarningDialog (警告弹窗，条件渲染)
```
