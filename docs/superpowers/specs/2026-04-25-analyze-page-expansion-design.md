# 分析页扩展与"我的"页简化设计

**日期**: 2026-04-25
**状态**: 已批准

## 概述

将分析页从 2 页签扩展为 3 页签（增加学习路径页签），并将练习记录和错题本迁移为练习统计页签内的模态弹窗。同时简化"我的"页面，聚焦于学习设置。

## 问题背景

1. **分析页功能分散**：学习路径、练习记录、错题本分散在"我的"页面，用户需要在多个入口间跳转
2. **"我的"页面臃肿**：混合了数据展示、功能入口、设置等多种职责
3. **语义不清晰**：练习记录和错题本的业务含义需要在分析页中明确

## 设计方案

### 1. 分析页 - 三页签结构

```
┌─────────────────────────────────────────┐
│  学情解构                                │
│  深度解析掌握度与稳定性，锚定知识盲区     │
├─────────────────────────────────────────┤
│  图例：颜色深浅=掌握度 | 虚实=稳定性      │
├─────────────────────────────────────────┤
│  [成长分析] [学习路径] [练习统计]        │  ← TabSwitcher
└─────────────────────────────────────────┘
```

**TabSwitcher 配置**：
```typescript
const tabs = [
  { value: 'growth', label: '成长分析' },
  { value: 'path', label: '学习路径' },
  { value: 'practice', label: '练习统计' },
];
```

### 2. 各页签内容

#### 成长分析（默认显示，不变）

| 组件 | 说明 |
|------|------|
| 起始分校准提示 | 条件显示，当 `needsCalibration=true` 时 |
| 成长轨迹 | 首次/最近测评分数对比，成长值 |
| 数据可信度 + 波动范围 | 两个小卡片并排 |
| 知识掌握矩阵 | 柱状图 + 知识点卡片（诊断数据） |
| AI 学习建议 | 最多 3 条 |

#### 学习路径（新增页签）

**组件**: `LearningPathTab.tsx`

- 当前状态 + 进度条
- 本周统计（3 列网格：本周练习知识点 / 已掌握 / 待加强）
- 路径可视化
- 不包含编辑按钮和周报按钮（保持简洁）

**数据来源**: `/api/learning-path`

#### 练习统计（修改）

| 组件 | 说明 |
|------|------|
| 练习状态 | 正确率/总题数/分钟 |
| 本周练习趋势 | 折线图 |
| 知识练习分布 | 柱状图 + 练习知识点卡片 |
| 成就解锁 | 条件显示，当有成就时 |
| **练习记录入口** | 点击打开模态弹窗 |
| **错题本入口** | 点击打开模态弹窗 |

### 4. 模态弹窗设计

#### 练习记录模态

**Header**：
```

│ ← 关闭    练习记录           [开始练习] │

```

**内容**：练习记录列表（仅 `mode=training`）

#### 错题本模态

**Header**：
```
┌─────────────────────────────────────────┐
│ ← 关闭    错题本             [开始练习] │
└─────────────────────────────────────────┘
```

**内容**：错题列表（全部模式 diagnostic + training）

**数据来源**: `/api/practice/history`

**条目样式**：
```
┌─────────────────────────────────────────┐
│ ❌ 一元二次方程配方法                    │
│ 你的答案: x=2    正确答案: x=1 或 x=-½  │
│                                         │
│ [诊断测评] 错了3次 · 2024/01/15        │  ← 来源标签

```

**来源标签**：
- `[诊断测评]` - `mode=diagnostic`，背景：`bg-warning-container`
- `[日常练习]` - `mode=training`，背景：`bg-secondary-container`

**交互**：
- 点击关闭按钮 → 返回分析页
- 点击模态背景 → 返回分析页
- 底部导航保持在分析页（不可切换）
- Framer Motion 滑入/滑出动画

### 6. API 调整

**`/api/practice/history` 路由修改**：

```typescript
// GET /api/practice/history?mode=training
export async function GET(req: NextRequest) {
  const { searchParams } = new URL(req.url);
  const mode = searchParams.get('mode'); // 'training' | null

  const where: any = {
    userId: session.user.id,
    completedAt: { not: null },
  };

  // 仅在指定 mode 时过滤
  if (mode === 'training') {
    where.mode = 'training';
  }
  // mode 为 null 或其他值时返回全部记录
}
```

| API 调用 | 返回数据 | 用途 |
|---------|---------|------|
| `/api/practice/history` | 全部记录 | 错题本 |
| `/api/practice/history?mode=training` | 仅训练记录 | 练习记录 |

**错题本响应数据**：

每个 `attemptStep` 包含 `attempt.mode` 字段用于显示来源标签。

### 8. 实现顺序

1. **API 调整** - `/api/practice/history` 增加 `mode` 参数
2. **新增组件** - `LearningPathTab.tsx`
3. **修改组件** - `PracticeStatsTab.tsx` 增加入口按钮
4. **新增模态** - `HistoryModal.tsx`
5. **新增模态** - `MistakesModal.tsx`
6. **简化 MePage** - 移除功能入口，设置默认展开
7. **集成测试** - 验证页签切换和模态交互

### 10. 边界情况

| 场景 | 处理方式 |
|------|----------|
| 无学习路径 | 显示引导去测评 |
| 无练习记录 | 模态显示空状态 + 开始练习按钮 |
| 无错题 | 模态显示"暂无错题记录" |
