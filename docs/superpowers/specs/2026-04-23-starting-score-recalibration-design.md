# 起始分重新校准功能设计

**日期：** 2026-04-23
**状态：** 待实施

---

## 背景

### 问题
当前"学情解构"页面的起始分使用"历史最低分"。当用户试用时随意答题（如得10分），后来认真练习后（稳定80-100分），这个试玩低分会永久影响进步曲线的显示，不能真实反映产品对用户的帮助价值。

### 目标
为用户提供一个"救济渠道"，剔除试玩性质的无效数据，让进步曲线真实反映用户在认真使用产品后的提升。

---

## 功能设计

### 触发条件

当用户进入"学情解构"页面时，系统检测：
```
历史最低分 < 当前平均分 - 50
且 有效练习记录 >= 5 条
```

如果满足条件，显示智能提示卡片。

**定义：** "当前平均分" = 所有已完成练习（score > 0）的平均分

### 计算规则

**新起始分 = 去掉绝对最低分后，剩余分数中的最小值**

示例：
```
原始分数：[10, 85, 90, 95, 100]
- 绝对最低分：10
- 去掉10后：[85, 90, 95, 100]
- 新起始分：85
```

### 交互流程

1. **检测阶段**：用户进入"学情解构"页面
2. **提示阶段**：显示校准提示卡片（非模态，不阻断浏览）
3. **预览阶段**：用户点击"预览"，显示新起始分的计算结果
4. **确认阶段**：用户点击"确认"，执行校准
5. **完成阶段**：校准完成，更新显示，记录校准状态

### UI 设计

```
┌─────────────────────────────────────────┐
│  💡 检测到可能的试玩数据                 │
│                                         │
│  您的历史最低分（10分）与当前水平差异    │
│  较大，可能是试用时的数据。              │
│                                         │
│  重新校准后，起始分将变为：85分          │
│  进步幅度：+80分 → +5分                 │
│                                         │
│  [预览详情]  [保持现状]  [确认校准]     │
└─────────────────────────────────────────┘
```

### 产品原则

1. **一次有效**：校准后记录状态，不再重复提示
2. **用户主动**：校准需要用户明确确认，不自动执行
3. **透明预览**：让用户看到校准前后的变化
4. **可撤销**：用户可以撤销校准，恢复原始数据（可选）

## 技术实现

### 数据模型

**新增字段：** `User.startingScoreCalibrated` (boolean)
- `false`: 未校准，使用原始最低分
- `true`: 已校准，使用校准后的起始分

**新增字段：** `User.calibratedStartingScore` (integer?)
- 存储校准后的起始分
- `null` 表示未校准

### API 变更

**GET /api/analytics/overview**
```javascript
// 新增返回字段
{
  overview: {
    // ... 现有字段
    needsCalibration: boolean,  // 是否需要校准
    calibratedStartingScore: number | null,  // 校准后的起始分
    originalLowestScore: number,  // 原始最低分
  }
}
```

**POST /api/analytics/recalibrate-starting-score**
```javascript
// 请求
{ }

// 响应
{
  success: boolean,
  newStartingScore: number,
  oldStartingScore: number,
}
```

### 前端组件

**新增组件：** `StartingScoreCalibrationCard`
- 位置：`AnalyzePage` 顶部，在"成绩提升"卡片之前
- 条件渲染：`needsCalibration === true`

**状态管理：**
- `showPreview`：是否显示预览详情
- `isCalibrating`：校准中状态

## 验收标准

- [ ] 检测条件正确触发（最低分 < 平均分 - 50）
- [ ] 提示卡片正确显示计算结果
- [ ] 用户确认后，起始分更新为正确值
- [ ] 校准后不再重复提示
- [ ] 边界情况处理（只有1-2条记录时不提示）

## 后续优化

1. **撤销功能**：允许用户撤销校准，恢复原始起始分
2. **批量标记**：允许用户手动标记多条记录为"试玩数据"
3. **智能识别**：根据答题时长、正确率等维度自动识别试玩数据
