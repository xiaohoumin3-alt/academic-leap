# 学习进度与智能推荐修复设计

**日期:** 2026-04-24
**状态:** 草案

## 问题

### 当前问题
1. **学习进度需要手动设置** - 用户不知道该填多少，一直保持0
2. **智能推荐不执行** - 因为 `overwrite=false`，有知识点就不执行
3. **界面分支混乱** - "智能推荐"和"手动勾选"是两个独立分支，设计笨拙

### 用户期望
1. **学习进度自动计算** - 基于学期时间占比，无需手动设置
2. **智能推荐一键勾选** - 点击后自动勾选到当前进度，可在树中修改
3. **界面统一** - 只有一个知识点树，智能推荐只是快捷操作

## 设计方案

### 1. 学习进度自动计算

**数据模型扩展：**
```prisma
model User {
  // 新增字段
  semesterStart   DateTime?  // 学期开始日期
  semesterEnd     DateTime?  // 学期结束日期
}
```

**计算逻辑：**
```
学习进度 = (当前日期 - 学期开始) / (学期结束 - 学期开始) × 100%

边界情况：
- 当前日期 < 学期开始 → 进度 = 0%，提示"学期未开始"
- 当前日期 > 学期结束 → 进度 = 100%，提示"学期已结束"
- 未设置学期 → 进度 = 0%，提示"请设置学期时间"
```

**默认值：**
- 首次使用时，根据当前月份推断默认学期
  - 3-7月：春季学期（3月1日-7月15日）
  - 9-1月：秋季学期（9月1日-1月20日）

### 2. 智能推荐简化

**行为：**
- 点击"应用推荐" → 根据当前进度计算应勾选的章节 → 自动勾选所有知识点
- 不再有 `overwrite` 参数，直接执行勾选操作
- 用户可以在知识点树中手动修改

**API 简化：**
```
POST /api/user/knowledge/recommend
- 不需要 body 参数
- 计算当前进度
- 清除现有勾选
- 勾选到进度对应的所有知识点
```

### 3. 界面统一

**当前界面问题：**
```
┌─────────────────────────────┐
│ [智能推荐] [手动勾选]  ← 分支 │
├─────────────────────────────┤
│ 智能推荐内容                 │
│ 或                           │
│ 知识点树                     │
└─────────────────────────────┘
```

**新界面：**
```
┌─────────────────────────────┐
│ 学习进度: 45% [━━━━━░░░░░░]  │
│ [应用推荐] [设置学期]         │
├─────────────────────────────┤
│ 知识点树（始终显示）          │
│ ✓ 第1章                      │
│   ✓ 知识点1.1                │
│   ⬜ 知识点1.2                │
│ ✓ 第2章                      │
│   ...                        │
└─────────────────────────────┘
```

**交互：**
- 学习进度条自动计算显示（可点击"设置学期"修改）
- "应用推荐"按钮：一键勾选到当前进度
- 知识点树始终显示，可直接勾选/取消
