# 分析页页签重构设计

**日期**: 2026-04-25
**状态**: 已批准

## 概述

将分析页（app/analyze/page.tsx）的"学情解构"内容拆分为两个页签，使用顶部横向切换（iOS Segmented Control样式）。

## 问题背景

当前分析页将诊断测评数据和练习数据混在一起展示，内容较长，用户需要滚动查看。两种数据性质不同：
- 诊断测评：反映真实水平，变化较慢
- 日常练习：反映巩固情况，变化较快

拆分页签可以让用户更聚焦地查看所需数据。

## 设计

### 页签结构

```
页面标题：学情解构
页面副标题：深度解析掌握度与稳定性，锚定知识盲区

┌─────────────────────────────────────┐
│  ┌─────────────────────────────┐  │
│  │  成长分析  │  练习统计      │  │  ← iOS Segmented Control样式
│  └─────────────────────────────┘  │
└─────────────────────────────────────┘
```

### 页签内容

#### 成长分析页签（默认显示）

| 组件 | 说明 |
|------|------|
| 起始分校准提示 | 条件显示，当`needsCalibration=true`时 |
| 成长轨迹 | 首次/最近测评分数对比，成长值 |
| 数据可信度 + 波动范围 | 两个小卡片并排 |
| 知识掌握矩阵 | 柱状图 + 知识点卡片（诊断数据） |
| AI学习建议 | 最多3条 |

#### 练习统计页签

| 组件 | 说明 |
|------|------|
| 练习状态 | 正确率/总题数/分钟 |
| 本周练习趋势 | 折线图 |
| 知识练习分布 | 柱状图 + 练习知识点卡片 |
| 成就解锁 | 条件显示，当有成就时 |

### UI规范

| 元素 | 规格 |
|------|------|
| 页签位置 | 紧贴"学情解构"标题下方 |
| 页签样式 | iOS Segmented Control（圆角矩形，选中态高亮） |
| 页签文字 | 成长分析 / 练习统计 |
| 切换动画 | 轻量淡入淡出，约150ms |

### 交互行为

- **状态保持**：切换页签时保持已选中的知识点状态
- **数据加载**：所有数据在页面加载时一次性获取，页签切换无网络请求
- **底部按钮**："继续今日训练"按钮在两个页签都显示

### 数据分组

现有数据保持不变，按页签分组渲染：

| 数据源 | 使用页签 |
|--------|----------|
| `overview.diagnosticAttempts` | 成长分析 |

| `trainingKnowledgeData` | 练习统计 |
| `timeline` | 练习统计 |
| `overview.trainingAvgScore` 等 | 练习统计 |
| `recommendations.insights.achievements` | 练习统计 |

### 组件拆分

建议提取以下子组件：
- `TabSwitcher` - 页签切换器
- `GrowthAnalysisTab` - 成长分析页签内容
- `PracticeStatsTab` - 练习统计页签内容

### 不变部分

- Legend（颜色深浅/虚实透明说明）
- 起始分校准提示卡片（复用现有组件）
- 底部"继续今日训练"按钮
- BottomNavigation
- 数据加载逻辑
