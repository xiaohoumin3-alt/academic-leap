# 我的页面重构设计

## 背景

当前"我的"页面与"分析页"存在数据重复展示问题：
- 分析页定位：专业学习分析（趋势、图表、诊断）
- 我的页面定位：账户管理 + 简单统计

**问题：** 简单统计与分析页功能重叠，数据分散，用户困惑。

**决策：** 移除我的页面的学习统计，专注账户管理和路径管理。

---

## 设计目标

1. **职责清晰：** 我的页面 = 账户管理，分析页 = 学习数据
2. **简洁优先：** 避免信息过载，使用下钻交互
3. **路径保留：** 学习路径概览保留在我的页面（属于个人配置）

---

## 数据流

### 用户信息
- 来源：`/api/auth/session`
- 字段：name, email, avatar

### 学习设置
- 来源：`/api/user/settings`
- 字段：grade, selectedSubject, selectedTextbookId, targetScore
- 更新：`POST /api/user/settings`

### 学习路径
- 来源：`/api/learning-path`
- 字段：path, roadmap, weeklySummary

---

## 移除内容

| 移除项 | 原因 |
|--------|------|
| 学习统计卡片（练习次数/答题数/正确率）| 与分析页重复 |
| 独立的 LearningSettings 组件 | 合并到用户信息卡片 |

---

## 保留内容

| 保留项 | 原因 |
|--------|------|
| 用户信息 | 账户管理核心 |
| 学习路径概览 | 路径管理属于个人配置 |
| 功能入口（练习记录/错题本/管理后台）| 导航功能 |
| 退出登录 | 账户管理 |
| 周报弹窗 | 与学习路径相关 |

---

## 组件变化

| 组件 | 变化 |
|------|------|
| `MePage` | 简化，移除 stats 状态和统计卡片 |
| `LearningPathOverview` | 无变化 |
| `LearningSettings` | 改为弹窗模式，不再作为独立卡片 |

---

## API 无需修改

所有现有 API 保持不变：
- `/api/auth/session` - 用户信息
- `/api/user/settings` - 学习设置（GET/POST）
- `/api/learning-path` - 学习路径
- `/api/analytics/overview` - 学习数据（分析页使用）
