# 用户年级字段统一设计

**日期:** 2026-04-24
**状态:** 已批准

## 问题

User表存在两个年级相关字段，导致数据不一致和用户困惑：

| 字段 | 当前用途 | 问题 |
|------|----------|------|
| `grade` | 用户注册年级 | 用户卡片显示，与学习设置不同步 |
| `selectedGrade` | 学习年级 | 学习设置使用，造成混淆 |

## 目标

统一为单一 `grade` 字段，用户信息卡片和学习设置使用同一数据源。

## 设计方案

### 数据模型变更

**移除字段：**
- `selectedGrade` - 不再需要

**保留字段：**
- `grade` - 用户年级，唯一真实来源
- `selectedSubject` - 学习科目
- `selectedTextbookId` - 选择的教材

### UI设计

```
┌─────────────────────────────────────────┐
│ 👤 张三                                  │
│ 913993571@qq.com                          │
│ 🎓 9年级 · 数学    ← 从学习设置读取      │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 🎓 学习设置              ✏️             │
│                                         │
│ 当前选择：9年级 · 数学                  │
│ 知识点：0/0                             │
│                                         │
│ [更改年级和科目]                         │
└─────────────────────────────────────────┘
```

### 交互流程

1. **首次使用**：用户选择年级/科目/教材
2. **更改设置**：点击"更改年级和科目"打开选择器
3. **数据同步**：选择后更新 `User.grade` 和 `User.selectedSubject`
4. **显示同步**：用户卡片自动显示最新选择

### 组件变更

**app/me/page.tsx:**
- 从 `/api/user/settings` 获取学习设置
- 用户卡片显示 `selectedGrade` + `selectedSubject`

**components/LearningSettings.tsx:**
- API调用使用 `grade` 而非 `selectedGrade`
- 简化状态管理

## 实施文件

1. `prisma/schema.prisma` - 移除 selectedGrade 字段
2. `app/api/user/settings/route.ts` - 使用 grade 字段
3. `app/me/page.tsx` - 从学习设置读取显示
4. `components/LearningSettings.tsx` - 简化状态逻辑
5. 数据库迁移脚本
