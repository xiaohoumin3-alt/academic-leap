# QIE 复杂度迁移机制设计

> **目标：** 解决"简单题目无法预测复杂场景表现"问题，通过特征空间映射实现能力跨复杂度迁移

> **约束：** 无新数据、无实验、在现有 v2.0 架构内扩展

---

## 问题陈述

**核心问题：** 简单题的测评信号无法有效预测学生在复杂场景的表现

**问题维度：**
- **A.** 认知维度单一：题目未区分不同认知负荷
- **B.** 情境真实性差：题目与实际应用场景脱节
- **C.** 复杂度层次平铺：缺少能力梯度谱系

**现有基础（v2.0）：**
- 已有特征提取：`cognitiveLoad`, `reasoningDepth`, `complexity`
- 已有预测概率：`encodeAnswer()` 返回学习前概率
- 但这些特征未被用于构建复杂度谱系

---

## 核心设计

### 映射函数

**数学形式：**

```
P_complex = P_simple · exp(- w · ΔC)

其中：
P_simple: 学生在简单题的预测概率
P_complex: 预测的复杂题概率
w: 权重向量 (w₁, w₂, w₃)
ΔC: 复杂度差值向量 (Δc₁, Δc₂, Δc₃)
w·ΔC: 加权方向投影
```

**向量定义：**

```
ΔC = (
  ΔcognitiveLoad,      // 认知负荷差值
  ΔreasoningDepth,     // 推理深度差值
  Δcomplexity          // 结构复杂度差值
)

w = (
  w_cognitive,         // 认知负荷权重
  w_reasoning,         // 推理深度权重
  w_complexity         // 结构复杂度权重
)
```

**物理意义：**
- 不同维度的复杂度有不同的"惩罚强度"
- 保留方向信息（不是简单距离惩罚）
- 连续、可导、单调

---

### 在线校准机制

**校准信号来源：**
- `encodeAnswer()` 返回的预测概率
- 真实答题结果 `correct`

**误差定义：**

```
e = (correct ? 1 : 0) - P_complex
```

**权重更新规则（带门控）：**

```
if P_simple ≥ τ AND ΔC_i > 0:
    w_i ← w_i + η · e · ΔC_i
    w_i ← max(0, w_i)           // 防止负权重
    w ← w / sum(w)              // 归一化防爆炸
```

**参数：**
- `w₀ = (1/3, 1/3, 1/3)` - 无偏初始化
- `τ ≈ 0.7` - 门控阈值（防止低置信度样本污染）
- `η` - 学习率（需调优）

**门控条件的作用：**
- 防止 MLP 系统误差被错误归因为"复杂度惩罚"
- 只有当简单题预测足够确定时才用于校准
- 保证学到的是"能力迁移规律"而非"模型偏差"

## 验证闭包

| 维度 | 状态 | 说明 |
|------|------|------|
| 参数约束 | ✓ | wᵢ ≥ 0, Σwᵢ = 1 |
| 边界条件 | ✓ | ΔC → 0 ⇒ P_complex → P_simple |
| 可微性 | ✓ | ∇P 存在，可对齐 |
| 单调性 | ✓ | 任一维度 ↑ ⇒ P_complex ↓ |
| 可学习 | ✓ | 在线校准收敛 |
| 无破坏性 | ✓ | UOK 结构不变 |
| 信号隔离 | ✓ | 门控防止模型误差污染 |

---

## 实现边界

**不做：**
- 不引入新的外部数据
- 不进行离线训练
- 不改变 UOK 核心结构
- 不实现其他候选函数（线性衰减、分维惩罚、阈值模型）

**做：**
- 在 `UOK` 类中添加 `predictWithComplexityTransfer()` 方法
- 在 `MLState` 中添加权重向量 `w`
- 在 `encodeAnswer()` 中触发权重更新（带门控）
- 添加 `getComplexityTransferWeights()` 用于调试/监控
