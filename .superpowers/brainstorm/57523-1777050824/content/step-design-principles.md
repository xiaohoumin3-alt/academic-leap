# 步骤设计原则 - 系统性分析

## 问题根因

当前模板的步骤设计存在**三个核心问题**：

1. **步骤定义不清晰**：什么是"有效步骤"？什么是"无效引导"？
2. **粒度不一致**：有些模板拆得太细，有些太粗
3. **公式泄露**：题目直接给出公式，降低了思维要求

---

## 一、步骤的严格定义

### ✅ 有效步骤的判定标准

一个"有效步骤"必须**同时满足**以下条件：

| 条件 | 说明 | 示例 |
|------|------|------|
| **可操作** | 学生需要做某种输入/选择 | 输入数字、点击选项 |
| **可判分** | 系统可以自动判断对错 | 有明确答案 |
| **有思维含量** | 需要运用知识/方法得出结果 | 不是重复性操作 |

### ❌ 无效步骤的类型

| 类型 | 特征 | 示例 |
|------|------|------|
| **纯引导** | 只是文字提示，无需操作 | "理解题意" |
| **公式泄露** | 直接给出答案路径 | "使用 C=4a 公式" |
| **重复操作** | 和上一步完全相同 | 再次输入相同的数 |

---

## 二、步骤粒度的层次划分

### 根据题型确定粒度

| 题型 | 粒度原则 | 示例 |
|------|----------|------|
| **纯计算** | 按运算层级拆分 | (1/2)÷2 → 5-□ → (2/3)×□ |
| **几何计算** | 按推理路径拆分 | 识别公式 → 代入数值 → 计算结果 |
| **应用题** | 按建模路径拆分 | 设未知 → 列方程 → 求解 |
| **证明题** | 按逻辑节点拆分 | 条件1判断 → 条件2判断 → 综合判断 |

### 粒度的"黄金法则"

> **每一步应该是一个"可独立验证的思维节点"**

- 太粗：无法定位错误发生在哪个环节
- 太细：增加了不必要的操作负担
- 恰好：学生出错时，系统知道他在哪个节点卡住了

---

## 三、公式泄露问题

### 当前问题示例

```typescript
// ❌ 错误：直接给出公式
ui: {
  instruction: `使用勾股定理 c² = a² + b²`,
  hint: 'c² = a² + b²',
}
```

### 修复方案

```typescript
// ✅ 正确：让学生回忆公式
ui: {
  instruction: `直角三角形，a=${a}, b=${b}，求斜边 c²`,
  hint: '勾股定理是什么？',  // 提示方法，不直接给公式
}
```

### 公式处理的分层策略

| 难度级别 | 公式处理 | 理由 |
|---------|---------|------|
| Level 0-1 | 提示公式名称 | 学生刚学，需要回忆 |
| Level 2-3 | 不给公式 | 学生应该掌握 |
| Level 4-5 | 隐含公式 | 需要学生自己识别 |

## 四、不同题型步骤设计模板

### 计算题（纯数值运算）

**原则**：按运算顺序拆分

```
示例：(2/3) × [5 - (1/2) ÷ 2]

步骤1：(1/2) ÷ 2 = □     → 0.25
步骤2：5 - □ = □         → 4.75
步骤3：(2/3) × □ = □     → 3.17
```

**关键**：
- 每一步都是一个**独立的运算**
- 中间结果需要被记录（用于下一步）
- 最后一步是最终答案

---

### 几何计算题（勾股定理）

**原则**：按"公式 → 代入 → 计算"拆分

```
示例：直角三角形 a=6, b=8, 求斜边

步骤1：计算 c² = a² + b² → 100
步骤2：计算 c = √c²       → 10
```

**关键**：
- 步骤1是"应用公式"
- 步骤2是"开方计算"
- 两步都有明确的数值答案

---

### 应用题（方程建模）

**原则**：按"建模 → 求解"拆分

```
示例：甲乙两车...求乙车速度

步骤1：设未知 + 列方程（可选）
步骤2：求解方程
```

**问题**：步骤1如何判分？
- 如果让学生自己列式，很难自动判分
- 如果给出模板，又降低了难度

**建议**：
- 方程题可以考虑**单步**（只求最终答案）
- 或者用**选择题**形式（选择正确的方程）

---

### 证明题/判定题（YES_NO）

**原则**：按逻辑节点拆分

```
示例：判定是否为正方形

步骤1：四条边相等吗？          → 是/否
步骤2：给出角度信息了吗？      → 是/否
步骤3：能确定是正方形吗？      → 是/否
```

**关键**：
- 每一步都是**独立的判断**
- 前面的判断为后面的判断提供信息
- 最后一步是综合判断

---

## 五、当前模板的系统性问题

### 问题1：步骤数与题型不匹配

| 模板 | 当前步骤数 | 应该是多少 | 问题 |
|------|-----------|-----------|------|
| sqrt_concept | 1步 | 1步 | ✅ 正确 |
| pythagoras | 2步 | 2步 | ✅ 正确 |
| pythagoras_word_problem | 3步 | 2步 | ❌ 第一步无效 |
| pythagoras_folding | 3步 | 2步 | ❌ 第一步无效 |
| quadrilateral_perimeter | 2步 | 2步 | ✅ 正确 |
| square_verify | 3步 | 3步 | ✅ 正确（判定题） |

### 问题2：公式泄露

```typescript
// trapezoid_property.ts - hint 中直接给公式
hint: `周长 = 上底 + 下底 + 2 × 腰 = ${upperBase} + ${lowerBase} + 2 × ${leg}`
```

这等于直接告诉学生怎么计算，降低了思维要求。

### 问题3：步骤说明不清晰

当前 UI：
- `instruction` 和 `hint` 区分不明显
- 没有明确告诉学生"这一步要做什么"

## 六、系统性修复方案

### Phase 1: 定义步骤规范

创建 `lib/question-engine/step-guidelines.md`：

1. 有效步骤的判定标准
2. 不同题型的步骤粒度规范
3. 公式处理的分层策略
4. instruction/hint 的写作规范

### Phase 2: 模板审查与修复

审查所有模板，检查：
- [ ] 每一步是否可操作、可判分、有思维含量
- [ ] 是否存在公式泄露
- [ ] 步骤粒度是否合理

### Phase 3: UI 改进

- 明确区分 instruction 和 hint
- 添加步骤进度显示
- 突出当前步骤要做什么
